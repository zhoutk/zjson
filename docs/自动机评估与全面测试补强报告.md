## 2. 自动机模式是否“必须”

## 2.1 技术判断

从 JSON 语法性质看：

- 词法层（数字、转义）可以用 DFA；
- 结构层（对象/数组嵌套）天然是下推自动机（PDA）问题；
- 解析器常见实现是“递归下降（等价于隐式栈）”或“显式栈状态机”。

你当前实现已经具备：

- 明确的深度上限（`max_depth`）防止无限递归；
- 越界读取防护和错误位置上报；
- O(1) 追加与 O(1) key 查找（`lastChild` / `keymap`）后，性能瓶颈已大幅缓解。

因此，**“是否必须自动机”答案是：不必须**。原因：

1. **收益/风险比不优**：整套重写风险高，容易引入语义回归；
2. **现状性能可用**：Release 解析吞吐已处于较高水平（见第 6 节）；
3. **你的核心诉求是 API 简单好用（add 构建）**，不是极限流式解析器。

## 2.2 何时再上自动机

满足以下条件时再考虑第二阶段：

- 需要 SAX/流式解析超大文档（>100MB）且内存受限；
- 需要可证明的严格状态迁移表（审计/合规要求）；
- 需要 SIMD/分阶段解析（类似 simdjson 体系）追求 GB/s 级吞吐。

## 3. 对既有优化的复审结论

你在上一份报告中的大方向是正确的，尤其以下项非常关键且合理：

1. **`lastChild` 消除尾插 O(n²)**：显著提升大对象解析吞吐；
2. **惰性 `keymap`**：高频 key 访问从线性退化问题中解耦；
3. **拷贝/赋值资源语义修复**：避免泄漏和空指针风险；
4. **字符串转义完整化**：对 RFC 8259 兼容性提升明显；
5. **错误位置（line/col）上报**：可维护性与定位效率显著提升。

可继续关注的 trade-off：

- `keymap` 与 `lastChild` 让节点体积增大，可能轻微影响 `toString` / deep copy 缓存局部性；
- 对你这个“构建+解析并重”的库来说，这个 trade-off 是可接受的。

## 9. 第 5 轮实施内容

### 9.1 PDA 解析器（替换主实现）

**核心变更**：将 `parse_json(int depth)` 递归下降解析器完全替换为 `parse_json_pda()` 显式栈下推自动机。

设计要点：

1. **6 个显式状态**：`VALUE`, `OBJ_KEY_OR_END`, `OBJ_KEY`, `OBJ_COLON`, `OBJ_COMMA_OR_END`, `ARR_COMMA_OR_END`
2. **堆分配栈帧**（`std::vector<Frame>`）替代调用栈递归，消除深嵌套栈溢出风险
3. **RAII 内存守卫**（`Cleanup` struct）：任何错误退出路径自动释放栈上所有半成品容器
4. **`closeTop()` / `deliver()` 闭包**：统一值传递逻辑，减少代码重复
5. **移除死代码**：`get_next_token()` 和 `expect()` 已完全删除

性能对比（Release x64, MSVC）：

| 指标 | 第 4 轮（递归下降） | 第 5 轮（PDA） | 变化 |
|------|---------------------|----------------|------|
| Parse | 71–127 MB/s | 77–120 MB/s | 持平 |
| ToString | 217–417 MB/s | 198–416 MB/s | 持平 |
| Deep Copy | 112–430 MB/s | 100–434 MB/s | 持平 |

结论：PDA 替换未引入可测量的性能退化，同时获得了栈安全优势。

### 9.2 UTF-8 合法性策略开关

新增 `validate_utf8_bytes()` 静态函数（RFC 3629 完整校验）：

- 检查多字节序列长度合法性
- 拒绝 overlong 编码
- 拒绝 surrogate 范围（U+D800–U+DFFF）
- 拒绝超出 U+10FFFF 的码点
- 拒绝无效的 lead byte 和 continuation byte

新增 API：

```cpp
static Json ParseJsonStrictUtf8(const std::string& input, std::string& errMsg);
// 严格模式（不允许注释）+ UTF-8 字节级校验
```

`parse()` 内部签名扩展为 `parse(input, err, allowComments, validateUtf8)`。

### 9.3 JSONTestSuite 子集自动回归

新增文件：`tests/test_jsontestsuite.cpp`

覆盖：

- **y_ (MUST accept)**：50+ 个用例，覆盖结构、数字、字符串、Unicode、多字节 UTF-8、空容器等
- **n_ (MUST reject)**：80+ 个用例，覆盖非法结构（未闭合、多余闭合、尾随垃圾）、非法数字（前导零、`.e` 等）、非法字符串（无效转义、未转义控制字符、不完整 Unicode）、非法对象（非字符串 key、缺冒号、尾逗号）、strict 模式注释拒绝
- **UTF-8 校验**：10 个用例（有效/无效字节序列、overlong、surrogate、截断）
- **深嵌套 PDA 安全**：100 层成功、102 层失败、10000 层不崩溃
- **压力回环**：3000 次 parse→copy→move→toString→reparse
