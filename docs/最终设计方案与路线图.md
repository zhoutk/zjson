# ZJSON 最终设计方案 & 路线图

> 版本基准：本评审报告对应仓库当前 `src/zjson.hpp` 的状态（已完成数值序列化现代化、JSONTestSuite 全量接入、徽章发布）。
> 目标读者：维护者 / 贡献者 / 引入方。

---

## 1. 设计目标（不变量）

| 维度 | 目标 |
| --- | --- |
| 易用性 | 接口形态贴近 JavaScript 中使用 JSON 的直觉：`obj["a"]["b"]`、`arr[0]`、链式 `add(...)`、`obj.toString()`、`Json(str)` 反序列化。 |
| 部署 | 单头文件（`src/zjson.hpp`），除 C++ 标准库外**零依赖**。 |
| 语言基线 | C++17（`std::to_chars`、`<filesystem>`、`std::void_t`、结构化绑定等）。 |
| 正确性 | 完整通过 RFC 8259 / ECMA-404 的 [`JSONTestSuite`](https://github.com/nst/JSONTestSuite) `y_*` & `n_*` 用例。 |
| 鲁棒性 | 解析任意嵌套深度不会爆栈（显式栈 PDA）。 |
| 性能 | 解析/序列化在万级节点量级保持可比性能；插入 / 键查找已优化为 O(1) 摊销。 |
| 安全 | 严格 UTF-8 校验可选开启；不读未拥有内存；析构链表化释放避免递归爆栈。 |

## 2. 现状总览

### 2.1 接口契约（公开 API，保持稳定）

构造：`Json()`、`Json(JsonType)`、`Json(initializer_list<pair<string, Json>>)`、`Json(const string&)`（含**自动判别**：以 `{` / `[` 开头则视为 JSON 文本，否则视为字符串字面量）、`Json(const char*)`、模板数值/布尔/`nullptr_t`、拷贝/移动。

静态：`Json::FromFile(path)`、`Json::ParseJson(s, err)`（宽松，允许 `//`、`/* */` 注释）、`Json::ParseJsonStrict(s, err)`（严格 RFC 8259）、`Json::ParseJsonStrictUtf8(s, err)`（严格 + UTF-8 字节级校验）。

读取：`operator[](int)`、`operator[](const string&)`、`contains`、`getValueType`、`size`、`isEmpty`、`is*`、`getAllKeys`、`first`、`last`、`indexOf`、`slice`、`toString`、`toInt/Double/Float/Bool`、`toVector`。

写入：`add` / `push` / `push_back` / `push_front` / `insert` / `extend` / `concat` / `remove` / `removeFirst` / `removeLast` / `clear` / `pop_*` / `take` / `takes`。

迭代：`JsonIterator`（基于子链表 `child→brother`），暴露 `key()` / `value()`。

> **本次评审未改变任何公开签名**，仅删除/替换了内部辅助函数 `floatToCharStar`、常量 `MinValue` / `DecimalLength`。

### 2.2 内部数据结构（不变）

```cpp
class Json {
    Json* brother;     // 同级链表（cJSON 风格）
    Json* child;       // 子节点链
    Json* lastChild;   // 尾指针 → O(1) append
    mutable std::unordered_map<string, Json*>* keymap;  // 懒构建，O(1) 键查找
    Type type;
    std::string valueString;
    double valueNumber;
    std::string name;
};
```

要点：
- **侵入式链表**节点天然支持链式索引 `obj["a"]["b"]`，且支持单节点表示「值」也表示「容器头」的二象性。
- `lastChild` 让 `add/push_back` 由 O(n) → **O(1)**。
- `keymap` 懒构建于 `operator[](const string&)` / `contains` 首次调用；任何修改（`add` / `remove` / `clear`）会**主动失效**它，下次查询再重建——以一次 O(n) 换取后续无限次 O(1) 命中。

### 2.3 解析器

- **显式栈下推自动机（PDA）**：`JsonParser::parse_json_pda()`。状态机覆盖 `VALUE / OBJ_KEY_OR_END / OBJ_KEY / OBJ_COLON / OBJ_COMMA_OR_END / ARR_COMMA_OR_END`。最大嵌套深度 `max_depth = 100`，到上限直接 `fail`，不会调用栈溢出。
- **三种入口模式**：宽松（默认，允许注释）/ 严格（禁注释）/ 严格 + UTF-8 校验。
- **错误信息**：包含 `line / col`，便于定位。

### 2.4 序列化器 ✅（本轮重写）

- **删除** `floatToCharStar` 这段手写定点小数转换（精度差，最多 6 位且会丢有效数字）。
- **新增** `appendNumber(double, string&)`：
  1. `NaN / ±Inf` → 写 `null`（RFC 8259 禁止）；
  2. 整数快速路径：当 `v == floor(v)` 且能放入 `long long` 时用 `%lld` 输出，并正确保留 `-0`；
  3. 通用路径：优先使用 `std::to_chars(buf, end, v)`（C++17，最短回写，locale 无关），探测失败回落到 `snprintf("%.17g")`（IEEE-754 round-trip 安全）。
- **探测机制**：`detail::has_fp_to_chars<T>` 用 `std::void_t` 在编译期判断浮点 `to_chars` 是否可用，避免在老 stdlib 上编译失败。
- **效果**：序列化结果是「最短可往返」表示，所有数值满足 `Json(parse(serialize(x))).toDouble() == x`。

### 2.5 已落地的现代化条目

| 旧实现 | 新实现 |
| --- | --- |
| `floatToCharStar` + 手摇定点循环 | `std::to_chars` / `snprintf("%.17g")`，SFINAE 探测 |
| `Json::add` O(n) 追加 | `lastChild` 尾指针，O(1) 追加 |
| `operator[](string)` 深度搜索 | 懒构建 `unordered_map` keymap，命中 O(1) |
| 递归解析 → 易爆栈 | 显式栈 PDA，深度受 `max_depth` 保护 |
| 析构递归子链 | `deleteJson` 迭代沿 `brother` 释放 |
| 解析错误无定位 | `make_error()` 附 `line/col` |
| 仅基本 UTF-16 代理对 | 严格 UTF-8 字节序列校验（`ParseJsonStrictUtf8`） |

## 6. 设计原则备忘

为后续维护者保留：

1. **公开接口稳定优先**——任何重构都不应改变 `Json` 类已发布签名；新能力以重载/新方法/新静态工厂提供。
2. **节点即文档**——`Json` 同时承担「叶值」「容器头」两种角色；不要为了「优雅」拆成多种类型，那将破坏 `obj["a"]["b"]` 这种直观链式语义。
3. **懒计算优于预计算**——`keymap` 等加速结构必须能被修改路径「主动失效」，永远不假设它存在。
4. **解析与序列化对偶**——任何对序列化格式（数字、转义、空容器）的改动都必须有对应的解析往返测试。
5. **进 RFC 8259、宽 ECMAScript**——默认入口（`ParseJson`）允许注释这种「真实工程刚需」；严格入口（`ParseJsonStrict`）保证 RFC 一致性，用于对外协议。

## 7. 推荐的对外说辞（README 顶段）

> ZJSON 是一款 C++17 单头文件 JSON 库，零依赖，使用体验对齐 JavaScript：`Json obj("{...}")`、`obj["a"]["b"]`、`obj.add(...)`. 解析与序列化均通过 [JSONTestSuite](https://github.com/nst/JSONTestSuite) 全部 283 个 `y_/n_` 用例（严格 RFC 8259）；序列化采用 `std::to_chars` 最短可往返；解析器为显式栈下推自动机，深度安全；`unordered_map` 懒索引带来 O(1) 键查找。
